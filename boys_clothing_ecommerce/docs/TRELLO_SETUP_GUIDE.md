# Trello Board Setup Guide

## SRS Phase Board Setup

### Board Name
**Second-Hand Boys Clothing E-Commerce - SE SRS Phase**

### Lists Structure

1. **Backlog**
   - All identified tasks and requirements
   - Items not yet prioritized

2. **To Do**
   - Tasks ready to be worked on
   - Prioritized items from backlog

3. **In Progress**
   - Tasks currently being worked on
   - Maximum 2-3 items per team member

4. **Testing**
   - Completed tasks awaiting review/validation
   - Documentation review items

5. **Done**
   - Completed and validated tasks
   - Finalized documentation

### Sample Cards for SRS Phase

#### Backlog
- [ ] Research requirement analysis methodologies
- [ ] Design questionnaire for potential users
- [ ] Conduct comparative analysis of similar systems
- [ ] Draft functional requirements
- [ ] Draft non-functional requirements
- [ ] Write user stories
- [ ] Document ethical considerations
- [ ] Create wireframes

#### To Do
- [ ] Complete questionnaire distribution
- [ ] Analyze questionnaire responses
- [ ] Write SRS Introduction section
- [ ] Write SRS Overall Description section

#### In Progress
- [ ] Document functional requirements (FR1-FR95)
- [ ] Document non-functional requirements (NFR1-NFR38)

#### Testing
- [ ] Review SRS document completeness
- [ ] Validate requirement numbering
- [ ] Check IEEE template compliance

#### Done
- [x] Project proposal completed
- [x] Requirements analysis completed
- [x] Wireframes created

### Labels

- **Documentation** - Documentation tasks
- **Research** - Research activities
- **Review** - Items needing review
- **Critical** - High priority items
- **Low Priority** - Can be deferred

### Team Member Assignment

Assign cards to team members based on their roles:
- **Member 1:** Functional Requirements, User Stories
- **Member 2:** Non-Functional Requirements, Security
- **Member 3:** Requirement Analysis Methodology, Questionnaires
- **Member 4:** Ethical Considerations, Documentation Review

### Weekly Updates

Update the board weekly:
- Move completed items to "Done"
- Review and prioritize "Backlog"
- Update "In Progress" status
- Add new tasks as needed

### Screenshot Instructions

Take screenshots of your Trello board:
1. Full board view showing all lists
2. Detailed view of "In Progress" list
3. Completed "Done" list

Include these screenshots in the Appendix of your final report.
